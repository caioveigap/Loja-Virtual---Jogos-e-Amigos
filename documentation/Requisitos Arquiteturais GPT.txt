***********************************************************************************************************************
RNF (Requisitos não funcionais):
***********************************************************************************************************************

RNF.01 — Confidencialidade dos dados
O sistema deve manter confidenciais os dados pessoais e as informações de pagamento.

RNF.02 — Desempenho de busca
O sistema deve responder às consultas de busca por nome ou categoria em até 2 segundos.

RNF.03 — Disponibilidade
O sistema deve manter disponibilidade mínima de 99% do tempo.

RNF.04 — Concorrência
O sistema deve suportar acesso simultâneo de até 500 usuários sem degradação excessiva de desempenho.

RNF.05 — Desempenho em operações críticas
O sistema deve processar compras, downloads e publicações com baixa latência percebida pelo usuário e de forma consistente.

RNF.06 — Escalabilidade
O sistema deve ser escalável para comportar crescimento no número de usuários, jogos, avaliações, comentários e publicações.

RNF.07 — Capacidade de armazenamento
O sistema deve ser capaz de armazenar e recuperar até 1TB de dados relacionados a jogos, usuários, bibliotecas, comentários e reports.

RNF.08 — Integridade dos dados
O sistema deve garantir a integridade dos dados de bibliotecas, compras, avaliações e publicações.

RNF.09 — Adaptabilidade de interface
O sistema deve se adaptar adequadamente a diferentes tamanhos de tela em dispositivos distintos.

RNF.10 — Evolutividade e manutenibilidade
O sistema deve permitir correção e atualização de funcionalidades sem comprometer os dados existentes.

RNF.11 — Tolerância a falhas
O sistema deve ser tolerante a falhas nas operações, com capacidade de recuperação de dados.

RNF.12 — Conformidade legal e privacidade
O sistema deve atender aos requisitos legais e de privacidade aplicáveis ao tratamento de dados dos usuários.

RNF.13 — Tempo de carregamento da interface
O sistema deve carregar a página de exibição em até 1,5 segundo.

RA (Requisitos Arquiteturais):
***********************************************************************************************************************

RA.01 — Autenticação multifator
A arquitetura de autenticação deve suportar verificação em duas etapas para acesso às contas.

RA.02 — Proteção de credenciais
As senhas devem ser armazenadas com hash seguro e salt, nunca em texto puro.

RA.03 — Criptografia em trânsito
Toda comunicação entre cliente e servidor deve usar canais criptografados, como HTTPS/TLS.

RA.04 — Armazenamento protegido de dados sensíveis
Dados pessoais e informações de pagamento devem ser protegidos por mecanismos de controle de acesso e criptografia em repouso.

RA.05 — Otimização de busca
A arquitetura de dados deve empregar indexação e mecanismos de cache para suportar buscas rápidas por nome e categoria.

RA.06 — Alta disponibilidade e recuperação
A solução deve prever redundância, monitoramento e mecanismos de failover para sustentar disponibilidade elevada.

RA.07 — Escalabilidade horizontal
A arquitetura deve permitir crescimento por adição de recursos e/ou instâncias sem reestruturação completa do sistema.

RA.08 — Processamento desacoplado de operações críticas
Compras, downloads e publicações devem ser tratados por componentes ou fluxos desacoplados quando necessário, para reduzir latência e manter consistência.

RA.09 — Persistência com garantias transacionais
A persistência dos dados deve usar transações, restrições e validações no repositório para preservar a integridade.

RA.10 — Observabilidade e auditoria
O sistema deve registrar logs centralizados e trilha de auditoria para operações importantes, como login, compra, upload, report, moderação e alteração de privacidade.

RA.11 — Interface responsiva
A camada de apresentação deve ser desenvolvida com layout responsivo para diferentes tamanhos de tela.

RA.12 — Evolução segura do sistema
A arquitetura deve suportar versionamento, migrações controladas e compatibilidade com dados existentes durante atualizações.

RA.13 — Estratégia de recuperação de dados
O sistema deve possuir backups, restauração e, quando necessário, replicação para recuperação após falhas.

RA.14 — Validação em múltiplas camadas
As entradas do usuário devem ser validadas na interface, na aplicação e na persistência para reduzir erros e uso indevido.

RA.15 — Privacidade por projeto
A arquitetura deve incorporar mecanismos de consentimento, retenção mínima de dados e tratamento compatível com exigências legais.
